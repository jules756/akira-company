# Rubric — UX Designer

Scored weekly by Auditor. Each dimension 0–5.

### 1. Design consistency (0–5)
- **5** — All client-facing surfaces (dashboards, Notion public pages, magnet landing pages) pull from a documented design system in `life/areas/design-system.md`. No one-off styling.
- **3** — Design system exists, inconsistent application.
- **1** — Ad-hoc designs.
- **0** — No system.

### 2. Turnaround on design requests (0–5)
- **5** — Design asks from PM / Lead Magnet Creator / Head of Marketing delivered within 1–2 heartbeats. No aging backlogs.
- **3** — Some requests age >3 days.
- **1** — Chronic backlog.
- **0** — Unresponsive.

### 3. Voice-agent UX quality (0–5)
- **5** — Voice scripts (per-client) follow conversational UX principles: short turns, clear recovery paths, handoff-to-human when out of scope. Tested against common hospitality scenarios.
- **3** — Scripts work but rough edges.
- **1** — Awkward conversations.
- **0** — No voice UX standards.

### 4. Magnet visual quality (0–5)
- **5** — Every live magnet has on-brand visual treatment: landing page layout, result-page design, shareable hook image. Driven by `design-review` skill.
- **3** — Visual consistent but lacks polish.
- **1** — Ugly.
- **0** — No visual design.

### 5. Accessibility + Nordic-localization signals (0–5)
- **5** — Client-facing surfaces work in Swedish / English. Contrast, tappable targets, screen-reader hints considered. Nordic hospitality aesthetic (clean, confident, minimal).
- **3** — Readable but English-only or sloppy.
- **1** — Accessibility broken.
- **0** — Not considered.

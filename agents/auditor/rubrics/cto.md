# CTO RUBRIC

**Scoring scale**: 1-5 per dimension. Average.

## Dimensions

**Technical Planning & Architecture (weight 35%)**
- 5: Comprehensive plans with diagrams, edge cases, trust boundaries, clear decisions documented.
- 3: Solid but some gaps.
- 1: High-level or missing critical considerations.

**Code Review Quality (weight 30%)**
- 5: Paranoid structural reviews catching N+1, races, security issues, invariants. Actionable feedback.
- 3: Catches major issues.
- 1: Superficial reviews allowing problems through.

**Retro & Learning (weight 20%)**
- 5: Insightful weekly retros with clear patterns, action items, measurable improvements.
- 3: Basic retros.
- 1: No meaningful learning loop.

**Decision Documentation (weight 15%)**
- 5: All significant decisions logged with context/options/reasoning in project docs.
- 3: Some decisions documented.
- 1: No paper trail.

**Overall**: Average score. CEO comments prioritized.

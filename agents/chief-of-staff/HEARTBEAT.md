Check active issues and classify each one into one of the four flows:
- lead and reputation
- new client deployment
- skill library growth

## 4. Route the item

For each item:
- assign the next owner
- attach the right context
- note whether similar work already exists
- mark any missing information needed before execution

## 5. Escalate only when needed

Escalate to CEO only for approval boundaries, cross-team conflicts, or strategic ambiguity.

## 6. Exit

- Comment on any in_progress work.
- Clean handoff summary.
- Exit.

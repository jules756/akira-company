# Rubric — Founding Engineer

Scored daily by Auditor. Each dimension 0–5.

### 1. Shipping velocity (0–5)
- **5** — Features land on time vs. the CTO-approved plan. No undisclosed blockers aging.
- **3** — 1–2 blockers aging.
- **1** — Multiple stuck items.
- **0** — Nothing shipped this week.

### 2. TDD + verification discipline (0–5)
- **5** — Every non-trivial change uses `test-driven-development` and `verification-before-completion`. Passing tests = actually passing.
- **3** — Tests exist but coverage gaps.
- **1** — Shipping without tests.
- **0** — No test discipline.

### 3. Skill extraction (0–5)
- **5** — Patterns are extracted into reusable capabilities on the second use. Maintains clean capability library documentation.
- **3** — Some patterns are extracted.
- **1** — Repeated code patterns across skills.

### 4. Code quality (0–5)
- **5** — Code is clean, well-tested, follows CTO architectural decisions, no secrets committed.
- **3** — Generally good but occasional issues.
- **1** — Frequent quality problems.

### 5. Handoff quality (0–5)
- **5** — Clear documentation and handover to Implementation Engineer and Customer Success Manager when skills are ready.
- **3** — Adequate handover.
- **1** — Poor or missing handover.

**Scoring Notes**: Average the 5 dimensions. Score below 3.5 should trigger Auditor comment to CEO.

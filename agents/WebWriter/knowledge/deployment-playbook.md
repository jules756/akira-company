# Deployment Playbook — Web Operations

> Owned by Web Operations. SOPs for every deploy type. Updated as the team learns.
>
> **Last updated:** 2026-04-16 (seed)

## Stack

- **Repo:** `jules756/akira-agent-website` (separate from `jules756/akira-company`).
- **Framework:** Next.js 14+ (App Router).
- **CMS for blog:** Sanity (cloud-hosted) — chosen for collaborative editing + Vercel-native integration. (If switching to MDX-in-repo, update this section.)
- **Hosting:** Vercel.
- **Branch model:** `main` is production. Preview deploys on every PR.

## Required Vercel project setup

- Project linked to `jules756/akira-agent-website` GitHub repo.
- Auto-deploy on push to `main`.
- Preview deploys on every PR.
- Custom domain `akira-agent.com` configured.
- SSL automatic via Vercel.
- Environment variables (Vercel project settings, not committed):
  - `NEXT_PUBLIC_GA4_MEASUREMENT_ID`
  - `NEXT_PUBLIC_GTM_CONTAINER_ID`
  - `SANITY_PROJECT_ID`, `SANITY_DATASET`, `SANITY_API_TOKEN`
  - `BEEHIIV_API_KEY`
  - `TALLY_*` (if needed)

## Article publish — checklist

- [ ] Markdown content added (Sanity push or MDX commit)
- [ ] Schema (JSON-LD) implemented per SEO + AEO Specialist's brief
- [ ] OG metadata (title, description, image) set
- [ ] Internal links per brief's `internal_links` map
- [ ] Vercel build succeeds
- [ ] Preview deploy verified
- [ ] Push to main → production deploy
- [ ] Live URL returns 200 OK
- [ ] Schema validates via Rich Results Test
- [ ] OG metadata renders in social preview tools
- [ ] Internal links resolve
- [ ] GA4 page_view event fires (verified in real-time)
- [ ] Page submitted to Search Console for indexing
- [ ] Confirmation comment to SEO + AEO Specialist with live URL

## Landing page deploy — checklist

- [ ] Next.js page implemented from v0 component code
- [ ] Tally form embedded (or React form built)
- [ ] GTM events wired: `lp_view`, `lp_form_start`, `lp_form_submit_success`, `lp_outbound_click`
- [ ] Form submission tested end-to-end (lands in Tally, syncs to Beehiiv)
- [ ] Preview deploy verified
- [ ] Production deploy
- [ ] Smoke test: page renders, form submits, success event fires
- [ ] Confirmation to Lead Magnet Creator (or Web Designer)

## Rollback procedure

When a deploy breaks production:

1. Vercel UI → Deployments → find last good deploy → "Promote to Production."
2. Confirm production URL is back to last good state.
3. File post-incident note in `life/areas/deploy-incidents.md` with: time, what broke, how detected, rollback time, root cause.
4. Fix forward in a new branch + PR; never push directly to main during incident response.

## Search Console operations

## Friday discipline

- No risky production deploys after 2pm Friday.
- Hotfixes only.
- Major releases (new pillar pages, design refreshes) ship Tue–Thu.

## Common pitfalls

- **Sanity content not refreshing** — likely needs an Incremental Static Regeneration revalidate. Check `revalidate` config or hit the revalidate API.
- **GTM events not firing** — usually a CSP issue or a missing dataLayer push. Validate in GTM Tag Assistant.
- **OG image not appearing** — Vercel OG image generator needs explicit dimensions; cache may need bust via query param.
- **Schema not validating** — JSON-LD spec mistakes; use Rich Results Test before confirming the publish.
